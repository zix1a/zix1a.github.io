Java变量

局部变量：局部变量是在方法、构造函数或块内部声明的变量

参数变量

方法参数变量的值传递方式有两种：值传递和引用传递。

值传递：在方法调用时，传递的是实际参数的值的副本。当参数变量被赋予新的值时，只会修改副本的值，不会影响原始值。Java 中的基本数据类型都采用值传递方式传递参数变量的值。

引用传递：在方法调用时，传递的是实际参数的引用（即内存地址）。当参数变量被赋予新的值时，会修改原始值的内容。Java 中的对象类型采用引用传递方式传递参数变量的值。

a = 10, b = 20

成员变量声明在一个类中，但在方法、构造方法和语句块之外。

成员变量对于类中的方法、构造方法或者语句块是可见的。

静态变量的线程安全性

Java 中的静态变量是属于类的，而不是对象的实例。因此，当多个线程同时访问一个包含静态变量的类时，需要考虑其线程安全性。

静态变量在内存中只有一份拷贝，被所有实例共享。因此，如果一个线程修改了静态变量的值，那么其他线程在访问该静态变量时也会看到修改后的值。这可能会导致并发访问的问题，因为多个线程可能同时修改静态变量，导致不确定的结果或数据一致性问题。

为了确保静态变量的线程安全性，需要采取适当的同步措施，如同步机制、原子类或 volatile 关键字，以便在多线程环境中正确地读取和修改静态变量的值。

public：所有类可见

如果几个相互访问的 public 类分布在不同的包中，则需要导入相应 public 类所在的包。由于类的继承性，类所有的公有方法和变量都能被其子类继承。

protected： 对同一包内的类和所有子类可见(不能修饰类)

子类与基类在同一包中：被声明为 protected 的变量、方法和构造器能被同一个包中的任何其他类访问；

子类与基类不在同一包中：那么在子类中，子类实例可以访问其从基类继承而来的 protected 方法，而不能访问基类实例的protected方法。

辨析public和static

在Java中，静态变量 (static variable) 和公共变量 (public variable) 有着不同的用途和行为，尽管它们可能同时存在于一个类中。

  1. 静态变量 (static variable)
    定义：静态变量是用关键字 static 声明的变量。

作用域：静态变量属于类而不是类的实例（对象）。这意味着所有该类的实例共享同一个静态变量。

存储：静态变量在类加载时分配内存，并且在整个程序运行期间都存在。

访问：可以通过类名直接访问静态变量（ClassName.variableName），也可以通过对象访问，但通常通过类名访问更具可读性。

生命周期：静态变量的生命周期与类的生命周期一致，在类第一次被加载时创建，并在程序终止时销毁。

  1. 公共变量 (public variable)

定义：公共变量是用关键字 public 声明的变量。

作用域：公共变量可以是静态的或非静态的。如果是非静态的公共变量，那么它属于对象，每个对象有自己独立的副本；如果是静态的公共变量，那么它属于类本身，所有对象共享同一个变量。

访问控制：public 关键字意味着该变量可以被任何其他类直接访问，无论是通过对象还是通过类名（如果是静态变量）。

访问方式：可以通过对象或者类名直接访问公共变量。

总结

静态性：静态变量是与类关联的，而公共变量的访问权限由 public 修饰符控制，它们可以是静态的或非静态的。

共享性：所有对象共享同一个静态变量，而非静态公共变量是每个对象的独立副本。

访问：静态变量通常通过类名访问，公共变量可以直接访问。

c语言里静态变量和全局变量的区别

在C语言中，静态变量（static variable）和全局变量（global variable）的区别主要体现在作用域、存储期以及可见性等方面。

  1. 全局变量 (global variable)

定义：全局变量是在所有函数外部定义的变量，通常位于文件的顶部。

作用域：全局变量在定义它的文件中是可见的（如果没有使用 static 修饰）。可以在文件的任何位置访问，甚至可以在其他文件中通过 extern 关键字访问。

存储期：全局变量在程序开始执行时被分配内存，并在程序结束时释放内存。它们的生命周期贯穿整个程序运行期间。

可见性：如果一个全局变量没有用 static 修饰，那么它在整个程序的所有文件中都是可见的（假设用 extern 声明）。

  1. 静态变量 (static variable)

静态变量在C语言中有两种主要的用法：静态局部变量和静态全局变量。

静态局部变量（函数内的静态变量）

定义：静态局部变量是在函数内部使用 static 关键字定义的变量。

作用域：静态局部变量的作用域仅限于定义它的函数内，与普通的局部变量一样。但是，与普通局部变量不同，静态局部变量在函数调用结束后不会被销毁，而是保持其值直到下一次函数调用。

存储期：静态局部变量在程序开始时分配内存，并在程序结束时释放内存。它的生命周期与全局变量类似，但作用域受限于函数内部。

静态全局变量（文件作用域的静态变量）

定义：静态全局变量是在所有函数外部定义，并使用 static 关键字修饰的变量。

作用域：静态全局变量的作用域仅限于定义它的文件。即使在同一个项目中的其他文件也不能访问它。这种情况下，static 关键字限制了变量的作用域，防止它被其他文件访问.

存储期：静态全局变量与普通的全局变量一样，在程序开始时分配内存，并在程序结束时释放。

总结

全局变量：在整个程序中都可以访问（除非在某个文件中通过 static 限制其作用域）。

静态局部变量：仅在定义它的函数中可访问，但在多次函数调用之间保持其值不变。

静态全局变量：作用域仅限于定义它的文件，不会被其他文件访问到，且其生命周期与普通全局变量相同。
